# 博客系统（面试版项目说明）

## 一句话介绍

这是我独立完成的 Flask 博客后端项目，覆盖了用户注册登录、文章 CRUD、评论管理、JWT 鉴权、权限控制和统一错误处理，是我后端项目能力的基础版本。

## 项目目标

- 跑通完整的后端业务链路（用户 -> 文章 -> 评论）
- 建立 API 规范意识（统一响应、状态码、错误处理）
- 练习鉴权和权限控制（登录态 + 仅作者可修改/删除）

## 技术栈

- Python / Flask
- Flask-SQLAlchemy
- MySQL + PyMySQL
- JWT（PyJWT）
- Werkzeug 密码哈希

## 核心功能

- 用户模块：注册、登录、当前用户信息、修改密码
- 文章模块：创建、分页列表、详情、更新、删除
- 评论模块：创建、列表、更新、删除
- 鉴权模块：`@login_required` + Bearer Token

## 我重点做的点

### 1) 统一响应与异常

- 成功统一用 `success(message, data, status_code)`
- 失败统一用 `error(...)` 或自定义异常
- 使用全局错误处理器，把错误都转成 JSON

### 2) 权限控制

- 文章更新/删除：只允许作者操作
- 评论更新/删除：只允许评论作者操作
- 未登录访问受保护接口统一返回 401

### 3) 列表能力

- 文章列表支持分页参数
- 支持关键字过滤、作者过滤、排序字段与顺序
- 做了参数兜底，避免异常输入导致接口崩溃

### 4) 工程化细节

- 请求日志记录请求方法、路径、状态码和耗时
- 错误日志与普通日志分离
- 模块拆分清晰（auth/validators/responses/exceptions/logger）

## 典型接口（面试常问）

- `POST /api/users/login`：返回 JWT，后续接口通过 Bearer Token 鉴权
- `GET /api/posts`：分页 + 过滤 + 排序的组合查询
- `DELETE /api/posts/<id>`：鉴权 + 作者权限校验

## 我做过的优化

- 从 `Query.get()` 迁移到 `db.session.get()`，兼容 SQLAlchemy 新版本
- 调整异常类型和状态码，提升接口语义准确性
- 补充边界场景测试，保证“未登录/不存在/越权”三类错误行为稳定
